Manual Testing Project – Guru99 Bank Website



This project demonstrates a complete Manual Testing workflow on the Guru99 Bank website, simulating real-world QA practices used in software projects.

It covers:

Reviewing 4 versions of the SRS to track requirement changes

Designing detailed test cases

Building a Requirement Traceability Matrix (RTM)

Executing tests and reporting defects with clear reproduction steps

Performing regression testing to ensure website functionality and stability

Collaborating with the team to achieve full requirement coverage

This repository showcases hands-on skills in requirement analysis, test case design, bug reporting, regression testing, and applying industry-standard QA workflows for web applications.
